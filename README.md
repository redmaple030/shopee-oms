一、 運行環境與要求 (System Requirements)

1. 作業系統：
   - Windows 10 / 11 (建議)
   - Windows 7 (需安裝 Service Pack 1)
   - 兼容linux 但是在一些圖標可能會有所異常

2. 必要軟體：
   - 若您執行的是 .exe 檔：無需安裝任何程式，直接開啟即可。
   - 若您執行的是 .py 原始碼：建議安裝 Python 3.10 以上版本。

3. 辦公軟體：
   - 需安裝 Microsoft Excel、WPS Office 或 LibreOffice 以便查看生成的報表。


二、 檔案結構說明

資料夾內應包含以下檔案：
1. 銷售記錄系統.exe (或 main.py)  : 主程式
2. sales_data.xlsx               : 資料庫檔案 (首次執行會自動產生)
3. readme.txt                    : 本說明文件

sales_data.xlsx 包含兩個工作表：
- [銷售紀錄]：所有訂單的流水帳，包含買家資料、成本、銷售額、扣費明細與淨利。
- [商品資料]：您建立的商品清單、成本與分類設定。


三、 功能操作指南

(A) 分頁一：銷售輸入 & 訂單
--------------------------------------------------
這是您最常用的頁面，用於記錄每一筆出貨。

1. 【訂單基本資料】：
   - 日期預設為當天。
   - 勾選「填寫顧客/寄送資料」可輸入買家 ID、選擇寄送方式與取貨縣市。
   - 💡 技巧：寄送方式選「面交」時，地點會自動填入面交；縣市欄位可打字快搜（如打「台」找台北）。

2. 【加入商品 (購物車模式)】：
   - 在左側搜尋框輸入關鍵字（商品名或分類Tag）。
   - 點選下方列表的商品，系統自動帶入成本。
   - 輸入售價與數量，按下「加入清單」。
   - 可重複此步驟將多樣商品加入同一筆訂單。

3. 【費用設定與結算】：
   - 手續費率(%)：輸入平台抽成比例 (如 7.5)。
   - 額外扣費：若有運費補貼、活動費，請選擇原因並輸入金額。
   - 系統會自動將這些費用依照銷售金額比例，分攤到每樣商品的成本中，算出精確淨利。

4. 【送出】：
   - 確認金額無誤後，按下「確認送出並寫入 Excel」。

(B) 分頁二：商品資料庫管理
--------------------------------------------------
將新增與更新功能分流，避免操作失誤。

1. 【左側 - 新增新商品】：
   - 輸入分類 Tag (如：3C、美妝)、商品名稱與進貨成本。
   - 若商品名稱已存在，系統會阻擋並提示您去右側更新。

2. 【右側 - 維護既有商品】：
   - 搜尋並點選要修改的商品。
   - 可修改分類、名稱與成本。
   - 按下儲存後，系統會記錄「最後更新時間」，方便追蹤價格變動。

(C) 分頁三：關於開發者
--------------------------------------------------
顯示軟體版本、開發者聯絡方式及開源授權聲明。


四、 重要注意事項 (⚠️ 請務必閱讀)

1. 【關閉 Excel 檔案】：
   當您要按下「確認送出」或「儲存商品」時，請務必確認 `sales_data.xlsx` 已經關閉。
   若 Excel 檔案開啟中，程式無法寫入資料，會跳出錯誤訊息。

2. 【資料備份】：
   本程式為單機版，資料儲存在您電腦的 `sales_data.xlsx` 中。請務必定期備份此 Excel 檔案，以免電腦故障導致資料遺失。

3. 【勿手動修改 Excel 標題】：
   請勿隨意更動 Excel 第一列的標題名稱（如：日期、商品名稱...），否則程式讀取時可能會發生錯誤。

4. 【版本更新】：
   若您更新了本程式的新版本，建議先備份舊的 Excel 檔。若新版程式增加了新欄位，建議將舊 Excel 改名備份，讓程式重新生成新的格式，再手動將舊資料複製過去。
