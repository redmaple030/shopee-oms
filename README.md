## 📖 專案背景 (Project Background)

隨著電商平台手續費結構日益複雜，加上雲端服務存在帳號停權及資安疑慮，本專案旨在提供一套 **「本地優先 (Local-First)」** 的管理工具。結合 Python 高效數據處理能力與 Google Drive API，達成「資料個人掌握，備份雲端同步」的混合架構。

---

## ✨ 核心技術亮點 (Technical Highlights)

### 1. 🛡️ 工業級數據持久化與防禦 (Data Persistence & Safety)

- **萬用存檔引擎 (Universal Save Engine)**：
  - **原子性寫入 (Atomic Write)**：實作「臨時緩衝置換 (Shadow Paging)」機制。系統寫入時會先生成 `.tmp` 緩衝檔，確認寫入流完整後才執行 OS 層級的 `rename` 覆蓋，杜絕寫入中斷導致的檔案損毀。
  - **執行緒互斥鎖 (Thread Locking)**：針對 Google Drive 背景同步與本地儲存的併發衝突，實作 `threading.Lock()` 確保資料存取的排他性 (Exclusivity)，防止「競態條件 (Race Condition)」。
  - **標頭繼承算法 (Header Inheritance)**：針對非關聯式資料庫 (Excel) 限制，開發了自動補位邏輯。當訂單首行因退貨被刪除時，元數據 (Metadata) 會自動流轉至次位記錄，確保數據鏈不中斷。

### 2. 📈 高精度財務核算體系

- **Decimal 運算引擎**：全系統棄用 `float` 浮點數，改採符合 **ISO 60559** 標準的 `decimal` 模組進行十進位運算。
- **會計級四捨五入**：實作 `ROUND_HALF_UP` 策略與「無條件進位 (Ceiling)」定價邏輯，精確處理平台費率分攤與邊際利潤回推。

### 3. 🚚 供應鏈決策支援 (SCM Support)

- **動態 KPI 權重模型**：將廠商績效拆解為「品質、備貨、滿足、運輸」四個維度，允許管理者根據戰略目標（如：極速到貨 vs. 低採購成本）動態調整評分參數。
- **科學補貨模型 (ROP)**：整合「銷售速率 (Velocity)」與「總前置天數 (Total Lead Time)」，結合「安全庫存係數」自動推算補貨點，降低資金積壓與斷貨風險。

### 4. ⚙️ 系統強健性設計 (System Robustness)

- **數據清洗管線 (Data Scrubbing)**：實作 `pd.to_numeric(errors='coerce')` 與 `fillna` 雙重過濾機制，確保非法輸入（如文字填入金額欄）不會導致系統崩潰。
- **認證與權限隔離**：採用 SHA-256 加密鹽值 (Salt) 與 `secrets_config` 隔離機制。實作「優雅降級 (Graceful Degradation)」，確保在缺少私有配置時仍能以預覽模式運行。

## 🛠️ 技術棧 (Tech Stack)

- **核心語言**: Python 3.13 (實作物件導向開發)
- **GUI 框架**: Tkinter / Ttk (自定義樣式映射，解決 Windows 主題渲染衝突)
- **數據處理**: Pandas, OpenPyxl, Numpy
- **安全性**: Hashlib (SHA-256), OAuth 2.0 (Google Drive API)
- **開發範式**: 守護語句 (Guard Clauses)、非同步執行緒 (Asynchronous Threading)
